## 2. 关键认知 — Spring 全景类比图

| Java 生态 | LLM 应用生态 | 角色 |
|---|---|---|
| Spring Framework | **LangChain** | 核心积木 (LLM/Tool/Chain/Memory) |
| Spring StateMachine | **LangGraph** | 状态机/图编排 |
| Spring Boot Starter | **DeepAgents** | 约定优于配置脚手架 |
| `@SpringBootApplication` | `create_deep_agent(model, tools, prompt)` | 一行启动 |
| `spring-boot-starter-web` | `deepagents` 包 | starter 自动装配 |
| `@Bean ChatClient` | `ChatTongyi(model="qwen-plus")` | LLM 实例化 |
| `@Configuration` | `RunnableConfig` | 装配配置 |
| `@RestController` | LangGraph `StateGraph` 节点 | 接入点 |
| `@Aspect` (AOP) | `Callbacks` / LangSmith | 横切关注点 |
| Spring Data JPA | `VectorStore` / `Retriever` | 数据访问 |
| `@Transactional` | LangGraph `Checkpointer` | 状态持久化 |
| `application.yml` profile | `.env` + `LLM_PROVIDER` | 环境隔离 |

## 3. 核心心智模型

### 3.1 LLM 应用 = LLM + Tools + Memory + 控制流

```
Input ─→ Memory (检索过往)
       ↓
       Prompt (含历史 + 上下文)
       ↓
       LLM
       ↓ (decide)
       Tool? ──Yes──→ 执行工具 → 回到 LLM
        │ No
        ↓
       Output ──→ Memory (写回)
```

### 3.2 DeepAgents = 内置 4 件套

1. **Planning Tool** (`write_todos`) — LLM 自管 todo 清单
2. **Virtual File System** — 长上下文卸载到"虚拟盘"
3. **SubAgent** — 派生子 Agent, 上下文隔离
4. **Detailed System Prompt** — 长指令模板, 教 LLM 用上述 3 件

### 3.3 三层记忆

| 层 | 范围 | 存储 |
|---|---|---|
| 短期 | 单次 invoke | messages 窗口 |
| 会话 | 单 thread 跨 invoke | Checkpointer (SqliteSaver) |
| 长期 | 跨会话/跨用户 | 向量库 (PgVector / InMemoryVectorStore) |

### 3.4 三种 Agent 编排姿势

| 姿势 | 适用 | 代码量 |
|---|---|---|
| 纯 LangChain 手写循环 | 极致控制力 | ~150 行 |
| LangGraph StateGraph | 复杂多步 + 断点续跑 | ~180 行 |
| DeepAgents `create_deep_agent` | 标准 Agent 需求 | ~50 行 |

## 5. 下一步 — 进阶方向

### 5.1 生产化

- **可观测性**: 接 LangSmith, trace + eval + dataset
- **部署**: FastAPI + Docker + K8s, 接 LangGraph Cloud
- **监控**: Prometheus + Grafana 看 LLM 调用量/延迟/失败率
- **成本**: 加 token 计数 + 模型路由 (简单问题路由 qwen-turbo)

### 5.2 能力扩展

- **多模态**: qwen-vl-plus (图片理解) + qwen-tts (语音合成)
- **MCP**: 接 GitHub MCP / 数据库 MCP, 跨生态打通
- **Anthropic Skills**: Claude 4.7 的 skill 系统, 沉淀工作流
- **Long Context**: qwen-long 1M 上下文, 跑超长文档

### 5.3 业务场景

- **智能客服**: Ch4.2.2 意图识别 + Ch4.2.3 RAG + 转人工
- **代码助手**: 仿 Claude Code, 读写文件 + 跑测试 + git
- **数据分析**: pandas + matplotlib + 自动出报告
- **个性化推荐**: Ch4.2.1 长期记忆 + 用户画像

### 5.4 学术 / 进阶

- **微调**: LoRA 让 base model 适合你的业务
