## 1. 项目定位

| 维度 | 决策 |
|---|---|
| **教学受众** | 有 Java/Spring 背景 / 想转 LLM 应用开发的工程师 + Python 入门者 |
| **学完目标** | 用 Python + DeepAgents 独立交付一个**有规划、有记忆、能识别意图、能调工具、能编排子 Agent**的真实可用 Agent |
| **代码标准** | 每个脚本都能 `python xx.py` 直接跑；每个 lab 都有 `verify.sh` 一键验证 |
| **预算** | 阿里云通义千问（DashScope，新用户免费额度足够走完全程） |
| **节奏** | 每个里程碑一次 commit，commit message 含验证证据 |

## 2. 课程结构（4 章 · 11 个里程碑）

### Part 1 · 理论篇

#### Ch 1 · Python 基础（针对 Java 工程师快速上手）

| 脚本 | 主题 | 对照 Java |
|---|---|---|
| `01_hello_world.py` | 跑通第一个 Python | `public static void main` |
| `02_functions_and_types.py` | 函数、参数、类型提示 | 方法签名、泛型 |
| `03_classes_and_oop.py` | 类、继承、`@dataclass` | POJO、Lombok |
| `04_imports_and_packages.py` | 模块、包、`__init__.py` | package、import |
| `05_venv_and_pip.py` | 虚拟环境、依赖管理 | Maven/Gradle |
| `06_async_basics.py` | `async/await` | CompletableFuture |
| `07_fastapi_mini.py` | FastAPI 一个 RESTful 接口 | Spring Boot @RestController |

#### Ch 2 · LangChain 基础

| 脚本 | 主题 | 关键 API |
|---|---|---|
| `01_chat_basic.py` | LLM 调用、流式 | `ChatTongyi.invoke()` / `stream()` |
| `02_prompt_template.py` | Prompt 模板 + Few-shot | `ChatPromptTemplate` |
| `03_lcel_chain.py` | LCEL 链式表达式 | `prompt | llm | parser` |
| `04_tool_call.py` | 工具调用（function calling） | `@tool` + `bind_tools()` |
| `05_react_agent.py` | ReAct Agent | `create_react_agent`（langgraph 官方） |

#### Ch 3 · 框架对比 — 同任务三实现

任务："给一个研究问题（如『LangGraph vs DeepAgents 区别？』），Agent 自动联网搜索、综合、输出研究报告"

| 实现 | 文件 | 行数预估 | 看点 |
|---|---|---|---|
| **A. 纯 LangChain** | `01_langchain_only.py` | ~120 | 链式拼装、手动管理流程 |
| **B. LangGraph** | `02_langgraph.py` | ~150 | 状态机、节点、条件边 |
| **C. DeepAgents** | `03_deepagents.py` | ~50 | 三参数启动、自动 Planning + FS |

输出对比表（执行时长、代码量、可维护性、扩展性）。

## 6. 方法论双栈

### 6.1 OpenSpec（spec-driven development）

**三态机**：proposal → apply → archive

```
openspec/specs/<capability>/
└── spec.md         # 这个 capability 的规约 (must / should / could)
```

每个 lab 的关键功能对应一个 capability（如 `multi-layer-memory`、`intent-routing`、`subagent-orchestration`）。

### 6.2 Superpowers（4 阶段）

```
brainstorm → writing-plans → executing-plans → verification-before-completion
```

每个 lab 走完整流程：
1. **Brainstorm**：本设计文档已完成
2. **Plan**：每个 lab 的 README.md 即 plan
3. **Execute**：写代码
4. **Verify**：`verify.sh` 跑通

## 8. 多 Agent 协作策略

| 角色 | 工具 | 职责 |
|---|---|---|
| **主 Owner** | Claude（本会话） | 架构设计、code review、commit、整体把控 |
| **Code Architect** | `feature-dev:code-architect` 子 agent | 每章 lab 详细架构设计 |
| **Researcher** | `Explore` 子 agent + `WebSearch` | 调研 LangChain/DeepAgents 最新 API |
| **Code Reviewer** | `feature-dev:code-reviewer` 子 agent | 关键 lab 完成后 review |
| **Codex 协同** | `codex` skill | 二号 coder/反思者，生成对照实现 |

**协作流程**（每个 lab）：
1. 主 Owner 拉清单（设计 + spec）
2. 派 Architect 详细设计（必要时）
3. 主 Owner 写代码
4. 跑 `verify.sh` 自检
5. 派 Reviewer 二审（关键 lab）
6. Codex 三审（关键算法/最佳实践）
7. Commit + push

## 9. 验收标准（PUA 闭环纪律）

任意 commit 必须满足：
- [ ] 该 commit 引入的脚本都能 `python xx.py` 跑通（mock 或真调）

- [ ] commit message 含 `Verify:` 段落贴出验证输出

## 10. 风险 & 兜底

| 风险 | 兜底 |
|---|---|
| DashScope 限流 / 余额不足 | `LLM_PROVIDER=ollama` 切换本地模型 |
| `deepagents` 库 API 变动 | requirements.txt 锁版本范围 |

| Tavily 没 key | `duckduckgo-search` 免 key 备选 |
| Windows 兼容性 | verify.sh 提供 Bash 版 + PowerShell 注释 |
