## 三大流派分组 — 看清谁在解决什么

```
┌─────────────────────────────┬─────────────────────────────┬──────────────────────────────┐
│   ① OpenAI/Anthropic 派      │   ② DeepMind / 学院派         │   ③ 工程实战派 / 生态层        │
│   (NLP + Scaling 为主)      │   (RL + 搜索 + 世界模型)     │   (框架 / Pattern / 落地)     │
├─────────────────────────────┼─────────────────────────────┼──────────────────────────────┤
│ 1 Karpathy   LLM OS         │ 3 Hassabis   AGI 两步 / DT  │ 4 Schluntz   5 Pattern        │
│ 2 Lilian Weng 三件套        │                             │ 5 Andrew Ng  4 Pattern        │
│ 7 Brockman   4 段式 Prompt  │                             │ 6 Chase      5 Levels / CA    │
│ 8 Altman     时间表         │                             │ 10 Tobi      Context Engr     │
│ 9 Sutskever  Superint=Agt   │                             │                              │
└─────────────────────────────┴─────────────────────────────┴──────────────────────────────┘
```

**为什么这样分**:

- **① 派** 解决"模型本身的能力上限怎么爆发" — 视角是模型/数据/规模. 你在 Ch1-Ch2 学 LLM/Prompt/Tool 的时候, 用的是他们提供的 mental model.
- **② 派** 解决"模型不行的时候怎么靠搜索/规划补回来" — 视角是 RL/MCTS/推理. 你在 Ch7 (Reflection 进阶) 学到的 Tree-of-Thoughts 类方法, 来自这一派.
- **③ 派** 解决"工程师怎么把上面两派的成果落到业务" — 视角是 Pattern/框架/Cognitive Architecture. 你在 Ch3-Ch9 整个实战部分, 每一行代码都站在这一派的肩膀上.

**潜规则**: 三派不是对立的——是**分工**. 把 ① 的能力, 用 ③ 的 pattern 落地, 必要时引入 ② 的搜索. 一个完整的生产 Agent 必须三派都用上.

## Timeline — 思想发布的关键时刻

```
2017                                                                              2025-10
│                                                                                       │
├── 2017          Karpathy: Software 2.0 (medium 文章)
│                  → 看待 LLM 的根基世界观: 神经网络权重 = 新代码
│
├── 2023-06       Lilian Weng: LLM Powered Autonomous Agents (Lil's Log)
│                  → 三件套 (Plan/Memory/Tool) 成为 Agent 圈事实教科书
│
├── 2023-11       Karpathy: Intro to LLMs / LLM OS (YouTube 演讲)
│                  → 把 LLM 抬到内核地位, 上下文=RAM, 工具=系统调用
│
├── 2024-03       Andrew Ng: 4 Agentic Design Patterns (X 系列)
│                  → Reflection / Tool Use / Planning / Multi-agent 命名落定
│                  → 他点名 Reflection 是最被低估的, 直接带火该模式
│
├── 2024-12       Schluntz & Zhang (Anthropic): Building Effective Agents
│                  → Workflow vs Agent 二分法 + 5 大 Pattern
│                  → 整个 2025 年 Agent 圈最被引用的实战指南
│
├── 2024 同期     Harrison Chase (LangChain): Cognitive Architecture / 5 Levels
│                  → Own architecture, outsource infrastructure
│
├── 2025-01       Sam Altman: Three Observations (2025/26/27 三步走)
│                  → 给行业打 Agent 元年时间锚
│
├── 2025-02       Greg Brockman (OpenAI): 4-Part Prompt Framework
│                  → Goal/Format/Warning/Context Dump 成为生产 prompt 推荐结构
│
├── 2025-03       Hassabis (DeepMind): Gemini DeepThink (AlphaGo-inspired search)
│                  → 学院派标杆, 推理 = 搜索 + 规划
│
├── 2025-06       Karpathy + Tobi Lütke: Context Engineering 联袂背书
│                  → Prompt Engineering → Context Engineering 概念正式接棒
│                  → Phil Schmid: The new skill in AI is not prompting, its context engineering
│
└── 2025-10       Andrew Ng: DeepLearning.AI Agentic AI Course (vendor neutral)
                   → 用纯 Python 教 4 Pattern, 不藏在框架里
```

**怎么读这条 timeline**:

- 2017-2023 是**理论奠基期** — Karpathy + Lilian Weng 给后续 8 年的 Agent 工程定下心智模型
- 2024 是**Pattern 收敛期** — Andrew Ng + Anthropic + Chase 三家把杂乱的尝试收敛成可教学的 Pattern
- 2025 是**Agent 元年** — 工程实践 (Brockman 4 段式) + 概念升级 (Context Engineering) + 时代叙事 (Altman 三步走) 同时落地
- 2026+ 是**Harness Engineering 期** (社区共识) — 给自主 Agent 设计完整运行环境, 我们这本教程的下一阶段方向

## 如何用本图导航本教程

> 别从头到尾顺着读 — 按你的目标进.

**场景 1 · 我是 Java 工程师, 想最快搭一个能跑的 Agent**:

- 先读 [Karpathy LLM OS](references/big-names/01-karpathy.md) → 理解 LLM 不是 chatbot 是内核
- 再读 [Lilian Weng 三件套](references/big-names/02-lilian-weng.md) → Plan + Memory + Tool 心智图
- 直接进 [labs/ch04-1-quickstart-ui](../labs/ch04-1-quickstart-ui/) → 三参数启动 + Gradio UI

**场景 2 · 我要做生产级 Agent, 不知道选 Workflow 还是 Agent**:

- 读 [Schluntz/Anthropic](references/big-names/04-anthropic-schluntz.md) → Workflow vs Agent 二分法
- 读 [Chase 5 Levels](references/big-names/06-chase.md) → 别一上来就 L5, 先看清自主性谱系
- 跑 [labs/ch05-workflow-vs-agent](../labs/ch05-workflow-vs-agent/) (决策树) + [labs/ch06-workflow-patterns](../labs/ch06-workflow-patterns/) (5 Pattern 实现)

**场景 3 · 我的 Agent 输出质量不稳定**:

- 读 [Andrew Ng Reflection](references/big-names/05-andrew-ng.md) → 仅靠 Reflection 就能比升级模型更大提升
- 跑 [labs/ch07-reflection](../labs/ch07-reflection/) 4 个脚本看 Self-Critique / Generator-Critic / Tool-Grounded
- 进阶: 读 [Hassabis DeepThink](references/big-names/03-hassabis.md) → 多分支搜索 (ToT) 是 Reflection 的深度版

**场景 4 · 我要把 Prompt 写得像生产代码**:

- 读 [Brockman 4 段式](references/big-names/07-brockman.md) → Goal/Format/Warning/Context Dump
- 读 [Tobi Lütke + Karpathy Context Engineering](references/topics/context-engineering.md) → 上下文工程是新工种

**场景 5 · 我老板问我为什么要做 Agent (商业说服)**:

- 读 [Altman 三步走](references/big-names/08-altman.md) → 2025 = Agent 加入劳动力
- 读 [Sutskever Superintelligence is agentic](references/big-names/09-sutskever.md) → 行业最强科学家共识

**场景 6 · 我已经写了一堆 LLM 调用, 不知道这套架构能不能换框架**:

- 读 [Chase: Own Architecture, Outsource Infrastructure](references/big-names/06-chase.md) → 认知架构是护城河, 框架是 commodity
- 跑 [labs/ch09-cognitive-arch](../labs/ch09-cognitive-arch/) → 同业务 L1→L5 整套演化, 看清哪部分必须自己写
